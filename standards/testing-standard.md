# Testing Standard

Version: 1.0
Status: Final

---

# Purpose

Standar ini mendefinisikan proses pengujian perangkat lunak untuk memastikan sistem memenuhi Requirement, stabil, dan siap digunakan.

---

# Objective

- Memverifikasi Functional Requirements.
- Memvalidasi Non-Functional Requirements.
- Menemukan defect sedini mungkin.
- Meningkatkan kualitas perangkat lunak.

---

# Inputs

- Functional Requirements
- Non-Functional Requirements
- Software Architecture
- Source Code

---

# Outputs

- Test Plan
- Test Cases
- Test Execution Report
- Defect Report
- Test Summary

---

# Testing Levels

- Unit Testing
- Integration Testing
- System Testing
- User Acceptance Testing (UAT)

---

# Testing Process

1. Review Requirements
2. Create Test Plan
3. Design Test Cases
4. Execute Test Cases
5. Record Results
6. Report Defects
7. Retest
8. Close Testing

---

# Test Case Rules

Setiap Test Case harus memiliki:

- Test Case ID
- Objective
- Preconditions
- Test Steps
- Expected Result
- Actual Result
- Status

---

# Defect Rules

Setiap defect harus memiliki:

- Defect ID
- Severity
- Priority
- Description
- Steps to Reproduce
- Expected Result
- Actual Result
- Status

---

# Testing Quality Checklist

- Complete
- Repeatable
- Traceable
- Measurable
- Independent
- Verifiable
