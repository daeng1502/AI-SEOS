# Artifacts

Version: 1.0
Status: Final

---

# Purpose

Artifacts mendefinisikan seluruh keluaran (deliverables) yang dihasilkan selama proses software engineering.

Artifact merupakan hasil resmi dari setiap Task dan menjadi dasar bagi Task berikutnya.

---

# Principles

- Standardized
- Traceable
- Versioned
- Reviewable
- Reusable

---

# Artifact Categories

- Vision
- Requirements
- Business Process
- System Modeling
- Database Design
- UI/UX Design
- API Specification
- Source Code
- Testing
- Deployment
- Documentation

---

# Artifact Definition

Setiap Artifact harus memiliki:

- Purpose
- Owner Role
- Source Knowledge
- Related Tasks
- Version
- Status

---

# Artifact Rules

- Setiap Artifact memiliki satu Role utama sebagai pemilik.
- Artifact harus dapat ditelusuri ke Knowledge sumber.
- Artifact dapat direview dan direvisi.
- Artifact menjadi input bagi proses berikutnya.

---

# Inputs

- Knowledge
- Context
- Tasks

---

# Outputs

- Engineering Deliverables

---

# Constraints

Artifact tidak bergantung pada AI Provider maupun AI Orchestrator.

Artifact hanya merepresentasikan hasil software engineering.
