# Tasks

Version: 1.0
Status: Final

---

# Purpose

Tasks mendefinisikan unit pekerjaan software engineering yang harus diselesaikan untuk menghasilkan Engineering Artifact.

Task merupakan pekerjaan yang dapat direncanakan, dijalankan, dan ditinjau.

---

# Principles

- Atomic
- Traceable
- Measurable
- Repeatable
- Standardized

---

# Task Categories

- Requirement Analysis
- Requirement Validation
- Business Process Modeling
- System Modeling
- Database Design
- UI/UX Design
- API Design
- Backend Development
- Frontend Development
- Testing
- Documentation
- Review

---

# Task Definition

Setiap Task harus memiliki:

- Purpose
- Inputs
- Outputs
- Required Roles
- Required Skills
- Produced Artifacts

---

# Task Rules

- Setiap Task memiliki tujuan yang jelas.
- Setiap Task menghasilkan minimal satu output.
- Task harus dapat ditelusuri ke Knowledge sumber.
- Task dapat digunakan kembali pada berbagai proyek.

---

# Inputs

- Knowledge
- Context
- Workflow

---

# Outputs

- Engineering Artifact

---

# Constraints

Task hanya mendefinisikan pekerjaan software engineering.

Task tidak bergantung pada AI Provider maupun AI Orchestrator.
