# Change Management

Version: 1.0
Status: Final

---

# Purpose

Dokumen ini mendefinisikan proses pengelolaan perubahan terhadap Project, Knowledge, Workflow, Requirement, dan Engineering Artifact agar seluruh perubahan dilakukan secara terkendali, terdokumentasi, dan dapat ditelusuri.

---

# Principles

- Controlled
- Traceable
- Documented
- Reviewable
- Approved

---

# Change Lifecycle

1. Change Request
2. Impact Analysis
3. Review
4. Approval
5. Implementation
6. Verification
7. Documentation
8. Closure

---

# Stage Definition

## Change Request

Perubahan diajukan secara resmi.

## Impact Analysis

Menganalisis dampak perubahan terhadap Project.

## Review

Perubahan diperiksa oleh pihak yang berwenang.

## Approval

Perubahan disetujui.

## Implementation

Perubahan diterapkan.

## Verification

Perubahan diverifikasi.

## Documentation

Seluruh perubahan didokumentasikan.

## Closure

Perubahan dinyatakan selesai.

---

# Change Rules

- Setiap perubahan memiliki ID unik.
- Seluruh perubahan harus memiliki alasan.
- Seluruh perubahan harus melalui proses Review.
- Perubahan yang disetujui harus memiliki riwayat versi.
