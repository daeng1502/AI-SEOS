# Purpose

Dokumen ini mendefinisikan siklus hidup Requirement mulai dari identifikasi hingga Requirement dinyatakan selesai dan diarsipkan.

---

# Principles

- Traceable
- Reviewable
- Versioned
- Controlled
- Validated

---

# Lifecycle Stages

1. Identification
2. Elicitation
3. Analysis
4. Validation
5. Approval
6. Implementation
7. Verification
8. Maintenance
9. Retirement

---

# Stage Definition

## Identification

Mengidentifikasi kebutuhan awal stakeholder.

## Elicitation

Mengumpulkan Requirement dari stakeholder.

## Analysis

Menganalisis Requirement untuk memastikan kelengkapan dan konsistensi.

## Validation

Memastikan Requirement sesuai kebutuhan bisnis.

## Approval

Requirement disetujui stakeholder.

## Implementation

Requirement menjadi dasar Engineering Artifact.

## Verification

Memastikan Requirement telah diimplementasikan.

## Maintenance

Requirement diperbarui apabila diperlukan.

## Retirement

Requirement tidak lagi digunakan dan diarsipkan.

---

# Lifecycle Rules

- Setiap Requirement memiliki ID unik.
- Setiap Requirement memiliki sumber.
- Seluruh perubahan Requirement harus memiliki riwayat versi.
- Requirement yang telah disetujui menjadi baseline proyek.
